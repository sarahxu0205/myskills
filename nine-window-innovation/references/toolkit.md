# 产品经理专用工具包

## 快速启动模板使用说明

当用户尚未提供完整产品信息时，可以引导用户填写 `assets/quick-start-template.md` 中的模板。该模板包含产品基本信息、当前挑战、创新目标三个模块，能够帮助用户快速梳理关键信息，为九窗法分析奠定基础。

**使用场景**：

- 用户只说"帮我分析一下我的产品"，但未提供具体信息。
- 用户描述了部分挑战，但缺少产品画像。
- 用户希望获得更系统、更深入的分析。

**引导话术**：

```
为了给您做更精准的九窗法分析，建议您先填写以下信息：

[粘贴 quick-start-template.md 内容]

您可以直接复制以上模板，填写后回复给我。如果部分信息不确定，也可以先提供您知道的内容，我会基于已有信息先进行初步分析。
```

## 常见场景快速诊断

### 场景1：用户增长放缓

**重点分析窗口**：窗口5（产品现状）→ 窗口8（产品创新）→ 窗口6（行业现状）

**创新方向**：用户获取、留存优化、价值主张升级

**追问建议**：

- 当前增长放缓主要表现在哪个环节？（获客/激活/留存/推荐/变现）
- 核心用户群体的使用频率和时长是否有变化？
- 竞品的增长策略是否有明显变化？
- 行业是否出现了新的流量入口或用户习惯变化？

### 场景2：功能同质化严重

**重点分析窗口**：窗口4（功能现状）→ 窗口7（功能创新）→ 窗口1（功能演进）

**创新方向**：差异化功能、交互创新、技术升级

**追问建议**：

- 当前功能与竞品相比，差异化点在哪里？
- 用户最常用和最不满意的功能分别是哪些？
- 是否有新技术可以重新定义该功能？
- 历史上该功能是如何演进的？未来可能的突破方向是什么？

### 场景3：商业化效果不佳

**重点分析窗口**：窗口5（产品现状）→ 窗口8（产品创新）→ 窗口9（生态创新）

**创新方向**：商业模式创新、变现路径优化、生态合作

**追问建议**：

- 当前主要的变现方式是什么？收入结构如何？
- 用户的付费意愿和付费能力如何？
- 是否可以通过增值服务、订阅、交易佣金等方式拓展收入？
- 行业内是否有成功的商业化模式可以借鉴？

### 场景4：新产品方向探索

**重点分析窗口**：窗口6（行业现状）→ 窗口9（生态创新）→ 窗口3（行业演进）

**创新方向**：市场机会识别、需求挖掘、产品定位

**追问建议**：

- 目标行业的市场规模和增长趋势如何？
- 行业内存在哪些未被满足的痛点？
- 技术、政策、社会变化是否会催生新需求？
- 团队的核心能力和资源适合切入哪个细分领域？

## 引导提问/话术模板库

### 信息不足时

```
为了更好地运用九窗法为您分析，我还需要了解一些关键信息：

1. **产品基本信息**：您的产品名称、类型、核心功能是什么？
2. **目标用户**：主要服务哪些用户群体？
3. **当前阶段**：产品处于启动期/成长期/成熟期/衰退期？
4. **核心挑战**：当前面临的最主要问题是什么？
5. **关键指标**：DAU/MAU/转化率/收入等核心数据表现如何？

您可以先回答其中您最确定的几项，我会基于已有信息先做初步分析。
```

### 场景确认时

```
根据您的描述，我理解您当前最关注的是【用户增长放缓/功能同质化/商业化效果不佳/新产品方向探索】。

我将重点围绕以下窗口展开分析：
- 【窗口X】
- 【窗口Y】
- 【窗口Z】

如果我的理解有偏差，请随时纠正。接下来我将基于九窗法为您系统梳理创新机会。
```

### 机会深化时

```
您对方案【编号】感兴趣！我们可以进一步深入：

**深化方向：**
1. 详细设计：具体的功能模块和交互流程
2. 技术方案：实现该方案的技术路径
3. 商业模式：盈利模式和运营策略
4. 风险评估：潜在风险和应对策略
5. 原型设计：快速原型和测试方案

请选择深化方向（回复数字），或提出您希望深入的具体问题。
```

### 澄清模糊信息时

```
为了更好地帮助您，我需要澄清一点：

您提到的"【关键词】"，我理解您的创新目标是关于【推测1】，还是【推测2】？

同时，您希望解决的核心痛点是什么？（例如：效率问题、成本问题、体验问题等）
```

### 用户拒绝补充信息时

```
明白，我将基于您已提供的信息尽力生成最佳方案。请注意，由于信息有限，部分分析将基于合理推断，您可以在后续迭代中进行调整。
```

## 质量检查清单

分析完成后，请检查以下事项：

- [ ] 是否覆盖了所有 9 个窗口？
- [ ] 各窗口分析是否有具体数据、案例或用户反馈支撑？
- [ ] 创新机会是否来源于窗口分析？
- [ ] 每个创新机会是否标注了来源窗口？
- [ ] 优先级评估是否考虑了实际约束？
- [ ] 实施建议是否具有可操作性？
- [ ] MVP 设计是否明确？
- [ ] 成功指标是否可量化？
- [ ] 是否区分了"基于用户提供信息的分析"和"基于AI推断的补充"？

## 后续深化方向

如果用户希望进一步深化分析，可以提供以下方向：

1. **详细设计**：具体的功能模块、交互流程、信息架构。
2. **技术方案**：实现路径、关键技术选型、技术风险。
3. **商业模式**：盈利模式、定价策略、运营策略、增长飞轮。
4. **风险评估**：市场风险、技术风险、竞争风险、法规风险及应对措施。
5. **原型设计**：快速原型方案、用户测试计划、迭代策略。
6. **数据指标**：定义北极星指标、关键过程指标、A/B 测试方案。

## 输出质量自检问题

在最终输出前，反问自己：

1. 用户能否从输出中清晰看到九窗矩阵的全貌？
2. 每个创新机会是否有明确的窗口来源？
3. 优先级排序是否有清晰的评分依据？
4. 实施建议是否具体到可以执行的行动项？
5. 是否给用户提供了下一步行动的明确指引？
6. 是否避免了过度推断，明确标注了基于推断的部分？
