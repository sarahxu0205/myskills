# SCAMPER 引导提问模板库

## 使用说明

本文档包含 SCAMPER 创新过程中各阶段的引导提问模板。根据用户当前的信息完整度，选择对应的模板进行引导。

---

## 阶段 1：目标澄清

### 当用户仅提供模糊目标时

```
为了更好地帮助您，我需要了解几个关键信息：

1. 您想要创新的产品、服务或流程是什么？
   请用一句话简要描述（例如："我想优化我们公司的报销审批流程"）

2. 您当前遇到的核心挑战或痛点是什么？
   例如：效率低、成本高、用户体验差、同质化严重等

3. 您期望通过这次创新达成什么成果？
   例如：降低成本 20%、提升用户满意度、缩短处理时间等

4. 有哪些约束条件需要考虑？
   例如：预算、时间、技术栈、法规、目标人群等
```

### 当用户提供了目标但未明确痛点时

```
您提到想要创新【目标产品/服务/流程】，这很有价值！

为了更精准地帮您分析，能否分享一下：

当前这个领域存在什么关键痛点？
- 用户最常抱怨的问题是什么？
- 现有解决方案有什么不足？
- 您观察到的未被满足的需求是什么？

（如果您不确定，我可以基于常见场景进行初步分析，您可以在此基础上补充或修正）
```

### 当用户提供了目标和痛点时

```
很好！我已经了解了：
- 创新目标：【目标】
- 关键痛点：【痛点】

接下来，我将依次应用 SCAMPER 七个维度为您分析可落地的创新方案。
```

---

## 阶段 2：七维分析引导

### 替代（Substitute）

```
我们先从"替代"维度开始思考：

- 产品/服务中的哪个部分最可以被替代？
- 可以用什么新材料、新技术、新流程或新角色来替换？
- 替代后会带来什么具体好处？
```

### 结合（Combine）

```
接下来思考"结合"维度：

- 我们的产品/服务可以与什么结合创造新价值？
- 可以整合哪些功能、服务、场景或技术？
- 结合后如何解决现有问题或提升用户体验？
```

### 适应（Adapt）

```
第三个维度是"适应"：

- 产品/服务目前在哪些场景或人群中表现不佳？
- 需要适应哪些新的环境、用户或市场需求？
- 调整时需要保留哪些核心价值？
```

### 修改/放大/缩小（Modify）

```
第四个维度是"修改"：

- 产品/服务的哪些特性可以放大、缩小或调整？
- 尺寸、外观、功能、强度、时间、成本中哪些最值得修改？
- 修改后会带来什么新价值或新体验？
```

### 用于其它用途（Put to other purposes）

```
第五个维度是"用于其它用途"：

- 除了现有主要用途外，产品/服务还能用于什么？
- 可以拓展到哪些新场景、新行业或新用户群？
- 需要做哪些调整才能适应新用途？
```

### 删除（Eliminate）

```
第六个维度是"删除"：

- 产品/服务中哪些部分最没有价值或用户抱怨最多？
- 哪些功能、步骤、元素或信息可以删除？
- 删除后如何提升核心价值或简化体验？
```

### 重新安排/逆向（Rearrange/Reverse）

```
最后一个维度是"重新安排/逆向"：

- 哪些部分的顺序、布局或结构可以重新安排？
- 可以从哪些相反的角度思考当前问题？
- 改变后会如何影响效率或用户体验？
```

---

## 阶段 3：跨维度组合探索

```
现在让我们尝试将不同维度的想法组合：

- 哪两个维度的创意结合后可能产生 1+1>2 的效果？
- 这种组合能解决什么单一维度无法解决的问题？
- 组合后的方案有哪些独特价值？
```

---

## 阶段 4：评估排序引导

```
我们已经生成了多个创意，现在需要对它们进行评估排序。

请从以下维度为每个创意打分（1-5 分）：
- 创新性：是否新颖、有差异化？
- 可行性：技术、资源、时间是否允许？
- 用户价值：是否解决真实痛点？
- 成本效益：投入产出比如何？

或者我可以直接基于这些信息为您筛选 Top 3 优先方案。
```

---

## 阶段 5：落地与验证引导

```
对于优先方案，我们需要考虑落地实施：

- 下一步最小化行动是什么？
- 需要哪些资源（人力、资金、技术）？
- 可能遇到哪些风险，如何应对？
- 如何快速验证这个方案是否有效？
```

---

## 阶段 6：迭代深化

```
您对方案【编号】感兴趣！我们可以进一步深入：

深化方向：
1. 详细设计：具体功能模块和交互流程
2. 技术方案：实现路径和关键技术
3. 商业模式：盈利模式和运营策略
4. 风险评估：潜在风险和应对策略
5. 原型设计：快速原型和测试方案

请选择深化方向，或提出您希望深入的具体问题。
```

## 特殊情况处理

### 用户拒绝补充信息

```
明白，我将基于您已提供的信息尽力生成最佳方案。请注意，由于信息有限，部分分析将基于合理推断，您可以在后续迭代中进行调整。
```

### 用户希望跳过某个步骤

```
明白，我们可以灵活调整流程。

当前进度：【进度说明】
您的选择：跳过【步骤名称】

我将直接进入【下一步骤】，基于当前信息继续分析。
```

### 用户输入过于宽泛

```
您的问题比较宽泛，为了更好地应用 SCAMPER 方法，建议我们先聚焦一个具体的创新对象。

例如：
- 一个具体的产品功能
- 一项具体的服务流程
- 一个具体的用户场景

您希望从哪个具体对象开始分析？
```
