# Pre-Mortem Analyst

## 目的

假设项目或产品已经失败，然后逆向分析失败的原因。这种方法比传统风险评估更强大，因为它打破了乐观偏见，强迫你进行诚实分析。

## 为什么 Pre-Mortem > 风险评估

| 传统风险评估 | Pre-Mortem |
|-------------|------------|
| "可能会出什么问题？" | "项目失败了，为什么？" |
| 乐观偏见过滤答案 | 解放诚实分析 |
| 被动防御 | 主动进攻 |

**研究证明**：Pre-Mortem 能提升30%的风险识别率

## 什么时候用

- 在启动新项目或产品之前
- 评估某个产品方向是否可行
- 压力测试已有方案
- 产品上线后复盘
- 重大决策前的自我质疑

## 核心流程

### 步骤1：设定场景

> "现在是[未来某个日期]。这个产品/项目彻底失败了。"

常用时间设定：
- 产品："产品上线6个月后，完全失败了"
- 功能："这个功能发布后，用户完全不感兴趣"
- 方向："做了3个月后，发现方向错了"

### 步骤2：头脑风暴失败原因

列出10+个失败原因，**不要过滤**：
- 越荒谬越好
- 越极端越好
- 先数量，后质量

### 步骤3：分类

把原因归入四大类：

| 类别 | 常见原因 |
|------|----------|
| **人** | 关键人物离开、技能缺口、方向不一致、缺乏认同 |
| **流程** | 时间线太紧、范围蔓延、依赖问题 |
| **技术** | 无法扩展、集成失败、安全问题 |
| **外部** | 市场变化、竞品动作、政策法规 |

### 步骤4：评级

对每个风险评估：

- **可能性 (Likelihood)**：H/M/L
- **影响 (Impact)**：H/M/L

### 步骤5：预防措施

Top 3风险，每个必须有具体的预防行动

### 步骤6：定义预警信号

每个风险定义早期预警指标

## 常见失败原因清单

### 人员问题
- 核心人物离职或精力不济
- 团队技能无法匹配需求
- 团队对目标理解不一致
- 关键决策者不认同
- 团队士气低落

### 流程问题
- 时间线过于激进
- 范围不断蔓延
- 过度依赖外部团队
- 缺乏有效的反馈机制
- 决策链条太长

### 技术问题
- 系统无法扩展
- 第三方集成出问题
- 安全漏洞
- 性能问题
- 技术债务积累

### 外部问题
- 市场需求发生变化
- 竞争对手推出更强产品
- 政策法规调整
- 经济环境恶化
- 供应链中断

## 自我质疑清单

对于产品方向，特别追问：

**需求真实性**
- [ ] 用户真的有这个需求吗？
- [ ] 痛点够痛吗？用户忍忍就过去了？
- [ ] 需求是持续的，还是一次性？

**市场时机**
- [ ] 市场窗口是否正确？
- [ ] 太早还是太晚？
- [ ] 用户行为是否已经改变？

**竞争位置**
- [ ] 为什么你能赢？
- [ ] 护城河是什么？
- [ ] 竞争对手会怎么反应？

**商业模式**
- [ ] 用户愿意付费吗？
- [ ] CAC能否覆盖LTV？
- [ ] 单位经济学成立吗？

**执行能力**
- [ ] 团队有能力做出来吗？
- [ ] 资源够吗？
- [ ] 时间线现实吗？

## 结合其他工具

Pre-Mortem 与以下工具配合使用效果更好：

- **JTBD需求挖掘**：验证"失败原因第一条：用户根本没这个需求"
- **Discovery Interview Prep**：通过用户访谈验证预警信号
- **第一性原理拆解**：追问隐藏假设

## 常见陷阱

| 陷阱 | 症状 | 对策 |
|------|------|------|
| 只列显而易见的问题 | 所有人都能想到的原因 | 强制想10+个，包括荒谬的 |
| 风险无法落地 | "市场变化"这种空泛词 | 具体到可行动的描述 |
| 没有优先级 | 列出20个同等重要的风险 | 必须选出Top 3 |
| 没有预警信号 | 知道有风险但无法监测 | 每个风险定义具体指标 |
| 过于乐观 | "我们不会犯这种错" | 假设失败已经发生 |

## 使用示例

**场景1：验证新产品方向**
```
产品：AI写作助手
失败场景："产品上线3个月，完全没有人用"

失败原因：
- 用户其实不需要AI写作，自己写更快
- 目标用户（企业）有合规要求不能用AI
- 竞品太强，ChatGPT已经满足需求
- ...

Top 3风险：
1. 需求不真实 → 先做5个用户访谈验证
2. 竞品碾压 → 找差异化切入点
3. 合规问题 → 调研目标市场政策
```

**场景2：评估现有产品迭代**
```
功能：新用户引导流程
失败场景："新用户激活率只有5%，几乎没人完成引导"

失败原因：
- 引导步骤太多太复杂
- 用户价值主张不清晰
- 引导时机不对
- ...
```
