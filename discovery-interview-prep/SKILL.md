# Discovery Interview Prep

## 目的

通过系统的准备流程，设计有效的用户探索访谈，确保访谈产生可操作的见解，而不是确认偏见或表面级别的反馈。

## 什么时候用

- 开始产品探索，需要验证问题空间
- 重新定位现有产品，需要了解新市场
- 调查用户流失或退出原因
- 在动手开发前评估功能想法
- 为客户开发冲刺做准备

## 什么时候不用

- 用户测试原型（改用可用性测试）
- 大规模定量研究（用调查问卷）
- 已经明确知道问题所在，只需要验证解决方案

## 准备流程

### 步骤0：收集背景信息

在设计访谈前，先收集：

**针对现有或计划中的产品：**
- 问题假设或产品概念描述
- 目标客户细分（如果已知）
- 现有研究数据（支持工单、流失数据、用户反馈）
- 产品定位材料
- 关键假设

**针对调查现有问题：**
- 客户投诉和支持工单
- 关于客户为何离开的假设
- 客户转向的竞争替代方案

**如果探索新问题空间：**
- 类似产品或相邻解决方案
- 竞争对手材料、客户评论
- 社区讨论和论坛

### 步骤1：明确研究目标

回答：这次访谈需要了解什么？

| 目标类型 | 描述 | 适用场景 |
|----------|------|----------|
| 问题验证 | 确认问题存在且足够痛苦 | 新产品想法 |
| 待完成工作发现 | 了解客户试图完成什么、现有方案为何失败 | 产品战略 |
| 留存/流失调查 | 找出客户离开或不激活的原因 | 现有产品有流失问题 |
| 功能优先级排序 | 验证哪些问题/功能对客户最重要 | 路线图规划 |

### 步骤2：确定目标客户

回答：我要采访谁？

选择正确的人：
- **经常遇到问题的人**（高痛苦频率）
- **尝试过解决该问题的人**（了解失败原因）
- **目标细分市场中的人**（发现潜在需求）
- **最近遇到问题的人**（记忆犹新）

越具体越好！不要采访"所有用户"，要精准定位。

### 步骤3：明确约束条件

回答：访谈面临哪些限制？

| 约束类型 | 说明 |
|----------|------|
| 访问受限 | 只能访谈5-10人，需要快速出结果 |
| 现有客户群 | 有100+活跃客户，可以轻松招募 |
| 冷启动 | 没有客户，需要从零招募 |
| 仅内部人员 | 只能采访销售/支持团队（代理研究） |

### 步骤4：选择访谈方法

根据目标、用户和约束，选择合适的访谈方法：

#### 问题验证访谈（Mom Test风格）

**问法：**
- "告诉我你上次遇到这个问题的情况"
- "你尝试了什么？结果如何？"

**关键：** 问过去行为，不问假设

#### 待完成工作访谈

**问法：**
- "你当时想完成什么？"
- "你考虑了哪些替代方案？"
- "是什么让你选择了X？"

**关键：** 关注动机，不是解决方案

#### 切换访谈

**问法：**
- "是什么促使你寻找新解决方案？"
- "离开旧工具的'推力'是什么？"
- "尝试新工具的'拉力'是什么？"

**关键：** 理解竞争定位

#### 时间线/旅程地图访谈

**问法：**
- "带我回顾你第一次遇到这个问题的情况"
- "接下来发生了什么？"
- "你如何尝试解决它？"

**关键：** 揭示完整背景

## 访谈框架

### 开场（5分钟）

1. 自我介绍（简短）
2. 说明目的：学习，不是推销
3. 说明会做笔记，不会分享给第三方
4. 询问是否可以录音（可选）

**话术：**
> "我不是来卖东西的，就是想了解你的真实体验。你的反馈会帮助我们做得更好。"

### 核心问题（30-40分钟）

**问题验证必问：**
1. "能跟我讲讲你上次遇到[问题]的情况吗？"
2. "你当时是怎么处理的？"
3. "结果怎么样？"
4. "如果满分10分，那件事让你有多沮丧？"
5. "为什么是这个分数？"
6. "你试过什么其他方法吗？为什么？
7. "如果现在还没解决，是什么阻止你不去解决它？"

**避免的问法：**
- ❌ "你会想要一个能XX的功能吗？"（引导性）
- ❌ "你觉得这个产品怎么样？"（太开放）
- ❌ "如果有了这个功能，你会用吗？"（假设性问题）

### 结束（5分钟）

1. "有什么我应该问但没问到的吗？"
2. "还有其他想补充的吗？"
3. 感谢+说明后续

## 需要避免的偏见

| 偏见类型 | 症状 | 对策 |
|----------|------|------|
| 引导性问题 | "你不觉得这个功能很方便吗？" | 问行为，不问看法 |
| 确认偏见 | 只听支持自己假设的反馈 | 主动找反例 |
| 解决方案优先 | 一直在讨论解决方案 | 回到问题本身 |
| 采访错误的人 | 采访的是决策者而非使用者 | 精准定位用户 |
| 样本太少 | 1-2次访谈后就停 | 至少5-10人 |
| 不记录 | 聊完就忘 | 立即记录关键洞察 |

## 输出格式

```
【访谈计划】

研究目标：[你要了解什么]
目标用户：[精准描述]
访谈人数：5-10人
时间范围：2周内

【选择的访谈方法】
[方法名称]
理由：[为什么选择这个方法]

【核心问题清单】
1. [问题1]
2. [问题2]
3. ...

【需避免的偏见】
- [偏见1]
- [偏见2]

【成功标准】
- [标准1]
- [标准2]

【访谈后勤】
招募方式：[如何找到受访者]
安排方式：[如何安排时间]
综合方式：[如何整理洞察]
```

## 访谈记录模板

```
受访者：[编号/匿名标识]
访谈时间：
访谈者：

【背景】
- 职业/角色：
- 使用类似产品的经验：

【关键洞察】
1. [洞察1]
2. [洞察2]

【引用】（直接引用的原话）
- "[原话1]"
- "[原话2]"

【待验证假设】
- [假设1]
- [假设2]

【下一步】
- [需要进一步了解的问题]
```

## 关键原则

1. **聊过去行为，不聊未来假设**：人们说的和做的往往不同
2. **听痛苦程度**：知道有多痛，才能判断优先级
3. **别推销，要倾听**：你的目的是学习，不是说服
4. **深度优于广度**：5个深度访谈胜过20个浅层问卷
5. **立即记录**：访谈结束马上整理，否则会忘
