# HIT构思助手 - 引导提问模板库

## 使用说明

本文档包含HIT构思过程中各阶段的引导提问模板。根据用户当前的信息完整度，选择对应的模板进行引导。

---

## 阶段1：创新目标澄清

### 当用户仅提供模糊目标时

```
为了更好地帮助您，我需要了解几个关键信息：

1. **您想要创新的核心问题或产品是什么？**
   请用一句话简要描述（例如："我想创新一个智能水杯"）

2. **您希望解决的关键痛点是什么？**
   例如：用户经常忘记喝水、现有产品功能单一、使用体验不佳等

3. **您期望达成的创新成果是什么？**
   例如：提高用户饮水量30%、让饮水变得更有趣、解决特定场景的问题
```

### 当用户提供了目标但未明确痛点时

```
您提到想要创新【目标产品】，这很有价值！

为了更精准地帮您构思，能否分享一下：

**当前这个领域存在什么关键痛点？**
- 用户最常抱怨的问题是什么？
- 现有解决方案有什么不足？
- 您观察到的未被满足的需求？

（如果您不确定，我可以基于常见场景进行初步分析，您可以在此基础上补充或修正）
```

### 当用户提供了目标和痛点时

```
很好！我已经了解了：
- **创新目标**：【目标】
- **关键痛点**：【痛点】

接下来，为了激发更有价值的创意，我们需要引入一些跨领域的对比对象。
```

---

## 阶段2：对比领域推荐

### 动态推荐生成指南

**核心原则**：不要依赖固定的预设列表。根据用户的创新目标和关键痛点，实时分析并生成最相关、最有启发性的对比领域推荐。

**生成逻辑**：
1. **深度理解用户输入**：分析创新目标的本质特征（功能类型、用户场景、核心痛点、行业属性）
2. **跨域联想引擎**：基于目标特征，从以下维度思考可能的对比领域：
   - **自然仿生维度**：生物结构、生态系统、自然现象、进化机制
   - **社会系统维度**：组织架构、城市运行、文化传播、经济模式
   - **体验服务维度**：娱乐休闲、教育培训、医疗健康、餐饮服务
   - **技术工程维度**：物理原理、化学过程、数学模型、工程系统
   - **艺术人文维度**：音乐节奏、绘画构图、文学创作、哲学思想
   - **商业运营维度**：供应链管理、品牌营销、客户关系、平台经济
3. **筛选与匹配**：从联想结果中筛选出：
   - 与创新目标**跨域性最强**的领域（打破思维定式）
   - 有**明确成功特性**可借鉴的领域（确保启发价值）
   - **多样性覆盖**的领域（不同维度，避免同质化）
4. **生成推荐说明**：每个推荐领域需包含：
   - 领域名称
   - 核心可借鉴特性（1-2个）
   - 与创新目标的关联逻辑（为什么这个领域有启发）

**约束条件**：
- 绝不使用固定的预设列表
- 每个推荐都应有明确的关联逻辑说明
- 确保推荐的领域确实与创新目标有足够差异（跨域性检查）
- 如果用户提供了自己的对比领域，优先使用用户的领域

### 推荐话术模板

#### 当用户未提供对比领域时

```
基于您的创新目标【目标】，我为您动态推荐以下跨领域对比对象：

**推荐选项：**
1. 【领域A】（【核心特性】）→ 【关联说明：为什么这个领域对您的创新目标有启发】
2. 【领域B】（【核心特性】）→ 【关联说明】
3. 【领域C】（【核心特性】）→ 【关联说明】
4. 【领域D】（【核心特性】）→ 【关联说明】
5. 【领域E】（【核心特性】）→ 【关联说明】

**您可以选择：**
- 回复"使用默认选项"：我将基于以上领域继续分析
- 回复数字（如"1,3,5"）：选择特定选项
- 提供您自己的对比领域
```

#### 当用户提供的对比领域不足3个时

```
您提供的对比领域很有价值！为了获得更丰富的联想，建议再补充【N】个领域。

基于您的创新目标，我可以推荐：
1. 【领域A】（【核心特性】）→ 【关联说明】
2. 【领域B】（【核心特性】）→ 【关联说明】

或者您也可以提供自己想到的领域。
```

#### 当用户提供的对比领域与创新目标过于相关时

```
您提供的对比领域与目标问题相关性较高。HIT方法的核心价值在于跨领域联想——从看似无关的领域中发现突破性启发。

基于您的创新目标，我建议您也可以考虑以下更具跨域性的对比方向：
- 【领域A】（【核心特性】）→ 【关联说明】
- 【领域B】（【核心特性】）→ 【关联说明】
- 【领域C】（【核心特性】）→ 【关联说明】

这些领域虽然与您的目标看似无关，但其核心特性可能对您的创新有独特启发。

当然，如果您希望基于当前选择继续分析，我完全可以这样做。或者您也可以混合使用：保留部分现有领域，再补充1-2个跨域领域。
```

## 阶段3：特性关注点引导

### 当用户未提供感兴趣的特性时

```
为了更精准地分析对比领域与您的目标的关联，您希望重点关注哪些方面？

**可选维度：**
1. **功能特性**：核心功能和运作机制
2. **设计理念**：设计思想和用户体验原则
3. **技术实现**：关键技术和原理
4. **价值主张**：为用户创造的独特价值
5. **全部维度**：综合分析所有方面

您可以选择一个或多个维度（例如回复"1,3"），或回复"全部"进行综合分析。
```

### 当用户提供了特性关注点时

```
收到！我将重点分析对比领域的【特性维度】维度，并探索其与【创新目标】的关联。
```

## 阶段4：创新方向确认

### 当用户未提供创新方向/限制时

```
在生成创意方案之前，是否有以下需要考虑的因素？

**可选补充（不影响当前分析）：**
1. **创新方向**：您是否有特定的创新方向偏好？
   - 技术创新
   - 体验创新
   - 商业模式创新
   - 服务创新

2. **限制条件**：
   - 预算范围
   - 技术可行性要求
   - 时间周期
   - 目标用户群体

3. **业务边界**：
   - 必须坚守的核心业务
   - 不能触碰的红线
   - 必须兼容的现有系统

您可以提供这些信息以获取更精准的方案，或回复"继续"基于当前信息生成。
```

### 当用户提供了创新方向/限制时

```
了解！我将基于以下约束条件生成创意方案：
- 【约束1】
- 【约束2】

确保生成的方案符合您的业务边界和创新期望。
```

---

## 阶段5：迭代深化引导

### 当用户希望深化某个方案时

```
您对方案【方案编号】感兴趣！我们可以进一步深入：

**深化方向：**
1. **详细设计**：具体的功能模块和交互流程
2. **技术方案**：实现该方案的技术路径
3. **商业模式**：盈利模式和运营策略
4. **风险评估**：潜在风险和应对策略
5. **原型设计**：快速原型和测试方案

请选择深化方向（回复数字），或提出您希望深入的具体问题。
```

### 当用户希望调整对比领域时

```
您希望调整对比领域！当前使用的领域是：
- 【领域A】
- 【领域B】
- 【领域C】

**调整选项：**
1. 替换某个领域（回复"替换【序号】为【新领域】"）
2. 增加新领域（回复"增加【新领域】"）
3. 完全重新选择（回复"重新选择"）
4. 使用默认推荐（回复"使用默认"）
```

---

## 阶段6：输出确认与反馈

### 当用户接受方案时

```
很高兴这个方案对您有帮助！

**后续建议：**
1. 可以针对推荐方案进行原型验证
2. 可以邀请更多利益相关者评估方案
3. 可以进一步细化实施路径

如果您需要针对某个方案进行深化设计，随时告诉我！
```

### 当用户希望优化方案时

```
收到您的反馈！请告诉我：

1. **具体优化方向**：
   - 提高可行性
   - 增强创新性
   - 降低成本
   - 缩短周期
   - 其他：【请描述】

2. **不满意的点**：
   - 【请描述】

我将基于您的反馈重新生成优化方案。
```

## 特殊情况处理

### 用户表示"不知道对比领域"

```
完全理解！选择对比领域正是HIT方法的价值所在——帮您发现意想不到的联想。

我将基于您的创新目标，智能推荐最适合的对比领域。您只需要：
1. 确认我的推荐是否合适
2. 或告诉我您更感兴趣的方向

让我为您推荐...
```

### 用户表示"没有限制条件"

```
好的，这意味着我们可以进行更开放的创意探索！

我将生成多个方向的方案，涵盖技术创新、体验创新、商业模式创新等。您可以在看到方案后选择最感兴趣的方向。
```

### 用户希望跳过某个步骤

```
明白，我们可以灵活调整流程。

**当前进度：**【进度说明】
**您的选择：**跳过【步骤名称】

我将直接进入【下一步骤】，基于当前信息继续分析。
```
