### 1. 小程序性能优化技能 (miniprogram-performance-optimizer)

**功能描述：** 识别小程序代码中的性能瓶颈，并提供基于微信官方最佳实践的优化建议。覆盖启动性能和运行时性能两大核心领域。

**主要特性：**
- 代码包体积优化检测
- 启动性能分析与优化
- 运行时性能监控
- setData 调用优化建议
- 渲染性能提升方案

**使用场景：**
- 用户明确要求进行性能优化
- 小程序出现启动慢、卡顿、白屏等性能问题
- 代码审查时发现潜在性能问题
- 部署前进行性能检查

### 2. 快速想法验证技能 (rapid-idea-check)

**功能描述：** 快速判断一个产品想法是否值得被快速落地。帮助独立开发者、创业者在投入开发前快速验证想法的可行性，输出直接、犀利的判断结论和具体建议。

**主要特性：**
- 五维快速扫描分析框架（需求真实性、用户价值、竞争格局、商业化可行性、独立开发者适配度）
- 基于七宗罪人性逻辑判断需求真实性
- 俞军公式评估用户价值
- 独立开发者视角的落地可行性评估

**使用场景：**
- 用户说"帮我判断这个想法值不值得做"
- 评估产品想法是否靠谱
- 决定是否要开发某个小程序/产品
- 在投入开发前快速验证想法可行性

### 3. 用户探索访谈准备技能 (discovery-interview-prep)

**功能描述：** 规划用户探索访谈，准备有针对性的访谈问题，避免常见偏见。通过系统的准备流程，设计有效的用户探索访谈，确保访谈产生可操作的见解。

**主要特性：**
- 完整的访谈准备流程（收集背景、明确目标、确定用户、选择方法）
- 多种访谈方法（问题验证、JTBD、切换访谈、时间线访谈）
- 常见偏见识别与避免策略
- 访谈记录模板

**使用场景：**
- 开始产品探索，需要验证问题空间
- 调查用户流失或退出原因
- 重新定位现有产品，了解新市场
- 在动手开发前评估功能想法

### 4. JTBD 需求挖掘技能 (jtbd-demand-mining)

**功能描述：** 使用 Jobs-to-be-Done (JTBD) 框架挖掘用户真实需求。系统性地探索用户试图完成的功能性、社交性和情感性工作，识别真正的痛点和收益。

**主要特性：**
- 三层用户工作分析（功能性、社交性、情感性）
- 痛点与收益识别框架
- 按严重程度排序需求
- 5个为什么追问技巧

**使用场景：**
- 验证产品想法的需求是否真实
- 深入理解用户"雇用"产品的根本原因
- 产品上线后用户量不达预期，分析原因
- 重新定位产品或进入新市场

### 5. 预死亡分析师技能 (pre-mortem-analyst)

**功能描述：** 假设项目已经失败，逆向分析失败原因。这种方法比传统风险评估更强大，能打破乐观偏见，强迫进行诚实分析。

**主要特性：**
- 逆向思维分析失败原因
- 四大类风险分类（人、流程、技术、外部）
- 风险评级（可能性×影响）
- 预防措施与预警信号定义
- 自我质疑清单

**使用场景：**
- 项目启动前进行风险评估
- 验证产品方向是否可行
- 压力测试已有方案
- 重大决策前的自我质疑

### 6. 小程序审核自查技能 (miniprogram-audit-checker)

**功能描述：** 帮助开发者在小程序提审前进行自我排查，检查是否符合微信审核要求。提供全面的审核检查清单，覆盖账号信息、服务类目、页面内容、用户隐私等多个维度。

**主要特性：**
- 全面的审核检查清单（账号信息、服务类目、功能审核、页面内容等）
- 风险分级识别（高/中/低风险）
- 官方文档依据
- 系统化的排查工作流程

**使用场景：**
- 准备提交小程序审核时进行全面自查
- 审核被拒后排查原因并提供修复建议
- 了解微信审核规范

### 7. 应用评论分析专家技能 (app-review-analyst)

**功能描述：** 专业应用商店评论分析技能，利用 AppInsightMCP 工具分析 Apple App Store 和 Google Play Store 的应用评论数据。支持评论情感分析、问题识别与分类、功能需求挖掘、竞品评论对比、版本迭代评价追踪、用户画像分析和市场机会识别。

**主要特性：**
- 评论情感分析（情感分布、趋势、关键词）
- 问题识别与分类（功能、性能、UI/UX、兼容性等）
- 功能需求挖掘（明确请求与隐含需求）
- 竞品评论对比分析
- 版本迭代评价追踪
- 用户画像分析
- 市场机会识别

**使用场景：**
- 分析应用商店评论和评分
- 竞品对比分析
- 挖掘用户需求和功能建议
- 追踪版本迭代后的用户反馈变化
- 产品决策前了解用户声音

### 8. Book to Skill 元技能 (book-to-skill)

**功能描述：** 将方法论书籍转换为可复用 Skill 的元 Skill。当用户上传 PDF 或 TXT 格式的书籍（尤其是方法论、工具书、指南类书籍）时，自动解析书籍内容，提取书名、作者、关键章节，并通过 16 个核心问题深度分析书籍内容，最终生成结构化的 Skill 文件。

**主要特性：**
- 自动解析 PDF/TXT 书籍文件
- 16 个问题框架深度分析（书籍定位、方法步骤、核心原则、适用场景等）
- 智能检测书籍类型（方法论 vs 非方法论）
- 自动生成标准化 Skill 文件结构
- 支持大书籍增量分析（分批处理）

**使用场景：**
- 用户说"把这本书变成 skill"
- 提取书籍的方法论和核心方法
- 将工具书转换为可复用的操作指南
- 从 PDF/TXT 书籍中提炼核心要点

### 9. AB 测试方案智能设计专家技能 (ab-testing-design)

**主要特性：**

- 科学的样本量计算（基于统计原理）
- 实验结果统计分析与显著性检验
- 多变量测试设计支持
- 数据驱动的决策建议

**使用场景：**
- 设计 AB 测试方案
- 计算实验所需样本量
- 分析实验结果的统计显著性
- 进行多变量测试设计
- 转化率优化实验

- 淡旺季平衡策略系统性规划
- 策略评分矩阵与组合策略设计
- 行业差异化策略指南（制造业、零售业、SaaS/软件、服务业、消费品）

### 11. HIT 启发式构思助手技能 (hit-ideation-assistant)

**功能描述：** 帮助用户通过跨域类比和特性转移，运用 HIT（启发式构思技术）方法生成突破性创意。通过比较看似无关的产品或服务，激发联想，形成创新方案，打破思维定式。

**主要特性：**
- 渐进式信息补全机制（基础级/进阶级/完整级）
- 创新目标解构与痛点分析
- 对比领域特性提取（功能、设计、技术、价值）
- 跨域联想与融合映射
- 3-5 个突破性创意方案生成
- 可行性/创新性/价值三维评估体系
- 创意优化与实施路径建议

**使用场景：**
- 用户需要创新产品或服务的创意
- 希望突破现有思维框架
- 需要从其他领域获取灵感
- 系统性生成和评估创意方案
- 解决复杂问题但缺乏思路

### 12. 播客内容提取器技能 (podcast-extractor)

**功能描述：** 从播客文字稿、音频转文字、访谈记录、会议纪要、节目笔记等长文本中提取结构化洞察，输出精美的单文件 HTML 页面（可本地直接打开）。消除填充内容、停顿和口语噪音，同时不遗漏任何有价值的信息。

**主要特性：**
- 元信息提取（播客名称、期数、发布日期、主持人/嘉宾、时长、主题标签）
- 内容摘要（一句话总结、核心论点、适合人群）
- 多维度结构化内容提取（核心观点、关键数据、案例故事、方法论框架、引用推荐、可执行建议、争议观点、时间戳映射）
- 支持 HTML（默认）和 Markdown（备选）两种输出格式
- 口语噪音过滤与质量检查
- 完整单文件 HTML 输出，无外部依赖，可直接本地打开

**使用场景：**
- 用户粘贴播客文字稿并要求分析/总结/提取
- 用户要求从播客内容中提取洞察、要点或结构化笔记
- 用户提及播客文字稿、节目笔记或音频转文字
- 用户想要"抓住要点"或"提取价值"
- 访谈记录、会议纪要的长文本结构化分析

### 13. 小程序 AI Skill 二轮优化技能 (wxa-skills-polish)

**功能描述：** 对 wxa-skills-generate 生成的小程序 AI SKILL 初稿进行标准化二轮优化。修复接口描述、Schema、组件绑定、API 返回值和组件 UI，补充官方最佳实践，使 SKILL 达到可发布状态。

**主要特性：**

- 补全 mcp.json 接口描述（语义化命名、字段级 description）
- 规范化 API 返回值（事实+动作两段式、错误处理、structuredContent 与 content 分离）
- 修复 API 模块导出与导入一致性
- 修复 SKILL.md 文档（业务流程图、接口依赖关系、业务约束、意图分流）
- 修复 setRelatedPage 调用与组件事件绑定
- 上行消息文案规范（用户视角、自然语言、简洁清晰）
- 打磨原子组件 UI（尺寸、背景、边距、字体、布局、操作）
- 验证与测试清单

**使用场景：**
- 用户手动调用优化生成的 SKILL
- 打磨 AI 技能使其达到发布标准
- 修复生成后的技能中的配置和代码问题
- 对 SKILL 进行标准化二轮优化

### 14. SCAMPER 创新思维专家技能 (scamper-innovation-expert)

**功能描述：** 运用 SCAMPER 七维法引导产品、服务或流程创新。通过结构化的七维检核表（替代、结合、适应、修改/放大/缩小、用于其它用途、删除、重新安排/逆向），系统性地发现改进机会，生成可落地的创新方案。

**主要特性：**
- 渐进式信息补全机制（基础级/进阶级/完整级）
- SCAMPER 七维全覆盖分析
- 每个维度产出 2-4 条具体创意
- 跨维度组合机会挖掘
- 四维度评估体系（创新性、可行性、用户价值、成本效益）
- Top 3 优先方案排序与落地建议
- 最小化验证方式与后续行动清单

**使用场景：**
- 需要对现有产品、服务、流程进行创新或优化
- 希望获得结构化的头脑风暴方向
- 面对创新瓶颈，希望打破思维定式
- 想要评估多个改进想法并排序优先级
- 需要跨行业借鉴经验并迁移到自身场景

### 15. 九窗法创新顾问技能 (nine-window-innovation)

**功能描述：** 运用九窗法帮助产品经理系统性地发现创新机会。通过时间维度（过去/现在/未来）与系统层级维度（子系统/系统/超系统）构建 3×3 分析矩阵，跳出单一视角，全面识别功能创新、体验优化、商业模式突破及跨界创新机会。

**主要特性：**
- 渐进式信息补全机制（基础级/进阶级/完整级）
- 3×3 九窗系统分析矩阵全覆盖
- 四类创新机会识别（功能创新、体验创新、模式创新、跨界创新）
- 五维度量化评估（用户价值、技术可行性、商业价值、竞争优势、资源匹配度）
- TOP3 高优先级机会详细分析
- MVP 设计、里程碑规划、资源需求与风险点识别
- 短期/中期/长期实施建议

**使用场景：**
- 产品处于增长瓶颈期，需要寻找第二增长曲线
- 功能同质化严重，需要差异化创新
- 商业化效果不佳，需要模式突破
- 新产品方向探索，需要市场机会识别
- 产品生命周期各阶段诊断与创新规划
