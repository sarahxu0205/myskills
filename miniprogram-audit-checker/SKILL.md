# 小程序审核自查助手

## 使用场景

1. **准备提交小程序审核时** - 进行全面自查，提前发现潜在问题
2. **审核被拒后** - 帮助排查被拒原因，提供修复建议
3. **想了解审核规范时** - 提供微信审核规则的详细解读

> **参考文档**：[微信小程序平台常见拒绝情形](https://developers.weixin.qq.com/miniprogram/product/reject.html)

---

## 审核检查清单

### 一、账号基本信息

#### 1.1 基本信息规范

小程序名称、简介、logo、服务范围、服务标签、账号基本信息文字均不得：

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| 侵犯他人权益 | 🔴 高 | 不得侵犯著作权、商标权、肖像权、名誉权等。不得使用不属于该小程序主体的品牌或商标、标识等内容或与之相似的内容、信息、特殊角标 |
| 违规广告用语 | 🔴 高 | 不得含有商业化用语的、热门小程序名称、"国家级"、"最高级"等新广告法明令禁止或其他无关的词语 |
| 违法违规内容 | 🔴 高 | 不得含有政治、色情、敏感、暴力血腥、恐怖、其他国家法律法规禁止的词汇及违法内容 |

**示例说明：**
- 检查名称、简介中是否含有该小程序不属于该账号的权益

---

#### 1.2 特别规则

##### 1.2.1 名称、简介

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| 简介明确功能点 | 🔴 高 | 简介需明确介绍小程序的功能点，不能使用模糊的词义表达。如不能使用"该小程序旨在提高用户的生活品味"这类模糊表达 |
| 名称简介一致性 | 🔴 高 | 名称、简介的信息表达的意思必须有关联，具有一致性，并应与实际提供的功能一致，不含有与功能无关的搜索热词 |
| 名称识别性 | 🔴 高 | 名称不能以电话、邮件、日历、小游戏等广义归纳类、斗地主、麻将等通用游戏名称类、或者其他普遍且不具有识别性词语来命名。名称必须是两个词以上的组合 |

**示例说明：**
- 能在简介中提炼该小程序的几个功能点
- 简介中能找到小程序名称或者分拆出来的词汇
- 当无法判断时，审核人员可主观判断

---

##### 1.2.2 头像logo

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| logo清晰度 | 🔴 高 | 小程序头像logo清晰度不够时，不予通过。无法看清、分辨、识别图片中包含的各个元素（如：文字、物体、形状等） |
| logo一致性 | 🟡 中 | 小程序头像logo应与名称、简介保持一致 |
| 官方品牌标识 | 🔴 高 | 小程序头像logo不得包含腾讯、微信等官方品牌标识 |

---

##### 1.2.3 服务范围和服务标签

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| 标签与范围一致 | 🔴 高 | 小程序所设置的服务标签，应与所选的服务范围保持一致。标签不能超出服务范围 |

**示例说明：**
- 服务范围是家政，服务标签不能是美食

### 二、服务类目审核

#### 2.1 类目与服务一致

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| 页面内容与类目一致 | 🔴 高 | 小程序服务类目所对应的页面中的核心内容必须与该类目一致 |
| 服务可直接使用 | 🔴 高 | 必须保证用户在该页面能使用该服务类目，不得隐藏，不得进行多次跳转 |
| 游戏类目限制 | 🔴 高 | 一旦选择了游戏类目，该类目将不可修改变更为其他小程序类目 |

---

#### 2.2 类目链接使用正常

小程序的服务类目链接使用正常，不存在违法违规或不符合与腾讯所签署的相关协议、腾讯公布的相关规则、规范等内容。

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| 链接可正常打开 | 🔴 高 | 小程序服务类目所对应的页面链接不能正常打开 |
| 无非法信息 | 🔴 高 | 小程序服务类目所对应的页面链接加载非法信息 |
| 无恶意广告 | 🔴 高 | 小程序服务类目所对应的页面链接加载恶意、色情广告 |
| 无侵权内容 | 🔴 高 | 小程序服务类目所对应的页面链接加载侵犯他人权益的内容 |
| 无违规词汇 | 🔴 高 | 不得含有商业化用语的、热门小程序名称、"国家级"、"最高级"等新广告法明令禁止或其他无关的词语 |
| 无违法内容 | 🔴 高 | 不得含有政治、色情、敏感、暴力血腥、恐怖、其他国家法律法规禁止的词汇及其他违法内容 |

---

### 三、小程序整体审核规则

#### 3.1 基本功能审核规范

| 检查项 | 重要性 | 检查要点 |
|--------|--------|----------|
| 功能与简介一致 | 🔴 高 | 小程序所实际提供的功能点，需与小程序的简介一致。功能包括但不限于简介中提炼的功能点 |
| 类目功能首页体现 | 🔴 高 | 小程序所提供的所有服务类目功能，必须在小程序首页得到体现，即在小程序首页必须能直达或者经过2次点击到达所有服务类目页面 |
| 服务范围合规 | 🔴 高 | 小程序实际所提供的服务不得属于尚未开放的服务范围。不应超出小程序平台已开放的类目库范围 |
| 隐藏/付费功能合规 | 🔴 高 | 小程序中若存在隐藏或付费功能（比如仅充值可见，仅会员可见等受限功能点），该功能的实现不得含有色情、暴力、政治敏感或其他违法违规内容，开发者提供的测试号需可完整呈现和体验该功能 |
| 功能使用价值 | 🔴 高 | 小程序的功能应具有使用价值，不能过于简单。示例：只有一个页面，只有一个按钮 |
| 不复制微信功能 | 🔴 高 | 未经腾讯公司授权的情况下，不得在小程序中提供与微信客户端功能相同或类似的功能。示例：小程序功能不能包含朋友圈、漂流瓶等 |
| 不展示第三方小程序 | 🔴 高 | 在未经允许或未经腾讯公司授权的情况下，不得展示和推荐第三方小程序。示例：不能做小程序导航，不能做小程序链接互推，小程序排行榜等 |
| 功能使用无前置条件 | 🔴 高 | 小程序功能的使用，无需以关注或使用其他号为条件。示例：使用A小程序时，必须同时使用B小程序 |

## 排查工作流程

当用户需要审核自查时，按以下步骤进行：

### 步骤1：了解小程序情况
询问用户以下信息：
- 小程序的主要功能是什么？
- 小程序名称和简介是什么？
- 头像logo是否清晰、合规？
- 选择了哪些服务类目？
- 服务标签是否与范围一致？
- 是否涉及用户数据收集（地理位置、通讯录等）？
- 是否有账号体系？
- 是否有付费/隐藏功能？
- 是否有广告内容？
- 是否涉及特殊内容（游戏、社交、电商等）？

### 步骤2：分类检查
根据官方文档结构，逐项检查：
1. **账号基本信息** → 检查名称、简介、logo、服务标签
2. **服务类目审核** → 检查类目与服务一致性
3. **基本功能审核** → 检查功能完整性、使用价值
4. **页面内容审核** → 检查诱导行为、广告、违规内容
5. **可用性和完整性** → 检查是否为完成品、有无严重Bug
6. **用户隐私和数据安全** → 检查数据收集授权、隐私保护
7. **技术实现规范** → 检查自动播放、退出选项等
8. **UI规范** → 检查浮层可关闭、不模仿系统通知等

### 步骤3：风险识别
根据检查结果，识别风险等级：
- 🔴 **高风险**：违反3.2.8违法违规内容、侵犯权益、诱导行为等，可能导致审核直接不通过
- 🟡 **中风险**：广告比例、UI细节等，可能需要修改
- 🟢 **低风险**：建议优化但不一定影响审核
